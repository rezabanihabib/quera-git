# Project Setup & Contribution Guide

## 🚀 راه‌اندازی پروژه

پس از کلون کردن پروژه، حتماً مراحل زیر را انجام دهید:

```bash
npm install
npm run dev
```

با اجرای دستور بالا، پروژه در حالت توسعه (development) اجرا خواهد شد.

---

## 🖥️ اجرای پروژه

برای مشاهده بهتر و سریع‌تر تغییرات، پیشنهاد می‌شود از **Live Server** استفاده کنید.

---

## 📦 قبل از Push کردن

قبل از ارسال تغییرات (push)، حتماً پروژه را بیلد بگیرید:

```bash
npm run build
```

این کار باعث می‌شود از صحت عملکرد کد در حالت production مطمئن شویم.

---

* هر تسک (Task) را می‌توانید به دلخواه انتخاب و انجام دهید.
* لطفاً قبل از شروع، مطمئن شوید تسک موردنظر توسط فرد دیگری در حال انجام نیست.

## 🔀 روند صحیح Contribution

❗ **به هیچ عنوان مستقیم روی برنچ اصلی (main / master) پوش نکنید.**

روند صحیح به این صورت است:

1. یک برنچ جدید ایجاد کنید:

   ```bash
   git checkout -b feature/your-feature-name
   ```

2. تغییرات خود را اعمال کنید و کامیت بزنید.

3. برنچ را پوش کنید:

   ```bash
   git push origin feature/your-feature-name
   ```

4. در نهایت یک **Pull Request (PR)** ایجاد کنید.

---

## 🧠 نکات مهم

* کامیت‌ها را با پیام‌های واضح و استاندارد بنویسید.
* از تغییر دادن کدهای دیگران بدون هماهنگی خودداری کنید.
* سعی کنید کدها تمیز، خوانا و قابل نگهداری (maintainable) باشند.
* در صورت وجود باگ یا ابهام، حتماً در PR توضیح دهید.
